AUTOMATED SETUP SCRIPT FOR YOU (To Ship with Each Device)
================================================================================

Name this file: setup-me-cam.sh
Ship it with each Pi or email to customers.

Save as: ~/setup-me-cam.sh
Make executable: chmod +x ~/setup-me-cam.sh

Content:

#!/bin/bash
set -e

echo "üîí ME_CAM - Secure HTTPS Setup"
echo "================================"
echo ""

# Get Pi's IP
PI_IP=$(hostname -I | awk '{print $1}')
echo "‚úì Your Pi's IP: $PI_IP"
echo ""

# Create HTTPS certificates
cd ~/ME_CAM-DEV
mkdir -p certs

if [ ! -f "certs/cert.pem" ]; then
    echo "üìú Generating SSL certificates..."
    openssl req -x509 -newkey rsa:4096 -nodes \
      -out certs/cert.pem -keyout certs/key.pem \
      -days 365 -subj "/CN=ME_CAM.com" 2>/dev/null
    echo "‚úì Certificates generated (valid for 365 days)"
else
    echo "‚úì Certificates already exist"
fi

# Add domain to local hosts
if ! grep -q "ME_CAM.com" /etc/hosts; then
    echo "$PI_IP   ME_CAM.com" | sudo tee -a /etc/hosts > /dev/null
    echo "‚úì Added ME_CAM.com to /etc/hosts"
else
    echo "‚úì ME_CAM.com already configured"
fi

# Restart service
echo "üîÑ Restarting service..."
sudo systemctl restart mecamera
sleep 3

if sudo systemctl is-active --quiet mecamera; then
    echo "‚úì Service running"
    echo ""
    echo "================================"
    echo "‚úÖ SETUP COMPLETE!"
    echo "================================"
    echo ""
    echo "Access your camera at:"
    echo "  https://ME_CAM.com:8080"
    echo ""
    echo "Default login: admin / admin"
    echo "‚ö†Ô∏è  Change your password in Settings!"
    echo ""
    echo "IP Address: $PI_IP"
    echo ""
    echo "Note: First access shows security warning."
    echo "Click 'Advanced' ‚Üí 'Proceed' (this is normal)"
    echo ""
else
    echo "‚úó Service failed to start"
    echo "Run: sudo journalctl -u mecamera -n 50"
fi

================================================================================
TROUBLESHOOTING GUIDE FOR CUSTOMERS
================================================================================

PROBLEM: "ME_CAM.com cannot be found"
SOLUTION:
  1. Make sure you added ME_CAM.com to your hosts file correctly
  2. Windows: C:\Windows\System32\drivers\etc\hosts
  3. Mac/Linux: /etc/hosts
  4. Format: [IP]  ME_CAM.com (with spaces between)
  5. Save file and restart browser
  6. Try: ping ME_CAM.com (should return Pi IP)

PROBLEM: "Not Secure" warning in browser
SOLUTION:
  This is NORMAL for self-signed certificates on local networks
  1. Click "Advanced"
  2. Click "Proceed to ME_CAM.com (unsafe)"
  3. You can ignore this warning

PROBLEM: Camera stream is black
SOLUTION:
  1. Check camera is connected to Pi
  2. Restart service: ssh pi@raspberrypi "sudo systemctl restart mecamera"
  3. Wait 10 seconds, refresh browser
  4. Still black? Run: ssh pi@raspberrypi "libcamera-still --list-cameras"

PROBLEM: Can't login to dashboard
SOLUTION:
  1. Check your username/password (default: admin/admin)
  2. Clear browser cache (Ctrl+Shift+Delete)
  3. Try incognito/private window
  4. Try different browser (Chrome, Firefox, Safari)

PROBLEM: Service keeps stopping
SOLUTION:
  1. SSH to Pi: ssh pi@raspberrypi
  2. Check logs: sudo journalctl -u mecamera -n 50
  3. Look for error messages
  4. Restart: sudo systemctl restart mecamera
  5. Contact support with the error log

PROBLEM: Out of disk space / slow recordings
SOLUTION:
  1. In dashboard: Click "Storage Details" modal
  2. Set retention days to 3-7
  3. Click "Clear Old Recordings"
  4. Check free space: df -h
